Набор Bash-скриптов для полной автоматизации развёртывания инфраструктуры в рамках демонстрационного экзамена. Скрипты запускаются на хосте Proxmox и управляют виртуальными машинами через `qm guest exec`.

> Скрипты покрывают задания **Модуля 1** и **Модуля 2**, чего достаточно для оценки «4».

---

## Архитектура

| Роль | Hostname | ОС | Задачи |
| :--- | :--- | :--- | :--- |
| Провайдер | `ISP` | Astra Linux | NAT, NTP, Nginx (Reverse Proxy + Web Auth) |
| Роутер HQ | `HQ-RTR` | Astra Linux | OSPF, NAT, VLAN, GRE, DHCP |
| Роутер BR | `BR-RTR` | Astra Linux | OSPF, NAT, GRE |
| Сервер HQ | `HQ-SRV` | Alt Server | RAID0, NFS, LAMP (Apache + MariaDB) |
| Клиент HQ | `HQ-CLI` | Alt Workstation | Ввод в домен, NFS-клиент, браузер |
| Сервер BR | `BR-SRV` | Alt Server | Samba DC (AD), Docker, Ansible |

---

## Как это работает

Главная особенность - использование `qm guest exec` для выполнения команд внутри VM без необходимости настроенной сети.

Для корректной передачи спецсимволов (`"`, `$`, `>`, пайпов) используется **Base64-инъекция**:
1. Команда формируется на хосте Proxmox
2. Кодируется в Base64
3. Передаётся в VM, где декодируется и исполняется через Bash

## Структура репозитория

```
demo2026/
├── env.sh          # настройки среды: VMID, имена интерфейсов
├── lib.sh          # вспомогательные функции (vm_exec, check_env, self_destruct)
├── 11_init.sh      # модуль 1.1 - базовая сеть, пользователи, SSH
├── 12_routing.sh   # модуль 1.2 - GRE, OSPF, DHCP, DNS
├── 21_xz.sh        # модуль 2.1 - Samba DC, RAID, NFS, Chrony, Ansible
└── 22_neebu.sh     # модуль 2.2 - Docker, LAMP, Reverse Proxy
```

`env.sh` и `lib.sh` - вспомогательные файлы, не предназначены для самостоятельного запуска.

## Требования

Перед запуском убедитесь, что:

- [ ] Proxmox VE версии 8.0 или выше
- [ ] QEMU Guest Agent установлен и включён на всех 6 виртуальных машинах
- [ ] Файл `Additional.iso` загружен в хранилище по пути `local:iso/Additional.iso`
- [ ] Все виртуальные машины запущены и агент отвечает на `qm agent <vmid> ping`

## Установка и запуск

### 1. Клонирование

Выполните в Shell Proxmox:

```bash
git clone https://github.com/arugva-lab/demo2026
cd demo2026
```

### 2. Настройка окружения

Откройте `env.sh` и укажите VMID и имена интерфейсов под вашу конфигурацию:

```bash
nano env.sh
```

Скрипты проверяют заполненность переменных при запуске и не продолжат работу, если что-то не указано.

### 3. Последовательный запуск

> [!IMPORTANT]
> Не запускайте все скрипты подряд. После каждого шага проверяйте результат вручную и заполняйте отчёт. Только убедившись что всё работает - переходите к следующему скрипту.

**Шаг 1 - Базовая настройка (Модуль 1.1)**

Timezone, сеть, VLAN, NAT, пользователи, SSH (порт 2026).
```bash
chmod +x 11_init.sh && ./11_init.sh
```

**Шаг 2 - Маршрутизация (Модуль 1.2)**

GRE-туннели, OSPF (FRR), DHCP, DNS (BIND).
```bash
chmod +x 12_routing.sh && ./12_routing.sh
```

**Шаг 3 - Сервисы и хранение (Модуль 2.1)**

Chrony, Samba DC, RAID0, NFS, Ansible.
```bash
chmod +x 21_xz.sh && ./21_xz.sh
```

**Шаг 4 - Веб и контейнеризация (Модуль 2.2)**

Docker (MariaDB + App), LAMP, Nginx Reverse Proxy, Web-аутентификация.
```bash
chmod +x 22_neebu.sh && ./22_neebu.sh
```

## Важные ограничения

> [!CAUTION]
> Скрипты **не идемпотентны** - повторный запуск без отката к снапшоту может сломать конфигурацию.

| Скрипт | Проблема при повторном запуске |
| :--- | :--- |
| `11_init.sh` | Дублирование записей в `/etc/network/interfaces` |
| `12_routing.sh` | Дублирование конфигов FRR, BIND, DHCP |
| `21_xz.sh` | Повторная инициализация Samba DC упадёт с ошибкой |
| `22_neebu.sh` | Docker compose поднимется поверх существующих контейнеров |

**Рекомендация:** делайте снапшоты всех VM перед запуском каждого скрипта.

## Чек-лист проверки

После выполнения всех скриптов убедитесь в следующем:

**Сеть и маршрутизация**
- [ ] Пинги ходят между HQ и BR, между VLAN 100 и VLAN 200
- [ ] На HQ-RTR и BR-RTR есть OSPF-соседи (`show ip ospf neighbor`)
- [ ] GRE-туннель `gre1` поднят на обоих роутерах

**Удалённый доступ**
- [ ] SSH на порту 2026 работает на всех машинах кроме ISP
- [ ] Баннер "Authorized access only" отображается при подключении
- [ ] Подключение под `net_admin` (роутеры) и `sshuser` (серверы) работает

**Файловый сервер**
- [ ] С HQ-CLI монтируется `/mnt/nfs` и можно создать файл
- [ ] Файл виден на HQ-SRV в `/raid/nfs/`

**Сервисы**
- [ ] Ansible: с BR-SRV выполняется `ansible all -m ping` успешно
- [ ] Домен: HQ-CLI входит под доменным пользователем (`hquser1`..`hquser5`)
- [ ] Сайты открываются по именам `web.au-team.irpo` и `docker.au-team.irpo`
- [ ] `web.au-team.irpo` запрашивает логин и пароль (WEB / P@ssw0rd)

## Почему это работает стабильно

- **Crontab `@reboot`** - восстанавливает сетевые интерфейсы, маршруты и сервисы после перезагрузки VM
- **iptables persistence** - правила сохраняются в `/etc/iptables.rules` и восстанавливаются при старте
- **MariaDB 10.11** - явно указанная версия вместо `latest`, которая может быть нестабильной
- **`systemctl enable`** - все сервисы настроены на автозапуск
